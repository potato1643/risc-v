# Объединённое покрытие — сводка (2026-06-03)

## Метод

Объединение трёх .info-файлов через `lcov --add-tracefile`:
- microtesk_coverage.info (MicroTESK RV64I, 30 ELF)
- riscof_rv64_coverage.info (RISCOF RV64I, 50 .S)
- imperas_rv32_coverage.info (Imperas RV32I, 48 .S)

Выполнено в Docker-контейнере `riscv-env` (lcov v1.14).

## Выводы

1. Объединённое покрытие трёх наборов — 16.9% строк, 13.3% функций
2. Прирост относительно лучшего отдельного (MicroTESK 16.2%) — всего 0.7 п.п.
3. Причина — высокое перекрытие (~92% покрытых строк затрагиваются ≥2 наборами)
4. MicroTESK (RV64I, алгоритмические) и Imperas (RV32I) дают наибольший
   уникальный вклад (~240 строк каждый)
5. RISCOF (RV64I, поинструкционные) почти полностью перекрывается MicroTESK (+78)
6. Важное ограничение: протестировано только расширение I (RV32I + RV64I).
   Расширения M, A, C, F, D (28+) не запускались на gcov-Spike.
