# MicroTESK: тестирование и сбор покрытия

## Обзор

MicroTESK генерирует тестовые программы из шаблонов (templates) на языке nML.
Тесты запускаются на симуляторе Spike, собранном с флагами `--coverage` для сбора покрытия gcov/lcov.

Spike запускается с ISA по умолчанию: `rv64imafdc_zicntr_zihpm`.
Все три расширения (I, M, A) тестируются на одном и том же бинарнике Spike — знаменатель lcov (53,227 строк) одинаков для всех.

## Пользовательский vs системный режим

Из всех расширений **только RV64I требует разделения на пользовательский/системный режим**.

| Расширение | Режим | Почему? |
|-----------|:---:|------|
| **M, A, C, F, D** | Только пользовательский | Операции над x-регистрами, без CSR |
| **Zicsr, Zifencei** | Только системный | CSR-операции, барьеры кэша |
| **MRET/SRET/WFI** | Только системный | Привилегированная архитектура |

**Правило:** `x0-x31/f0-f31` → пользовательский; `CSR/смена привилегий` → системный.

ECALL/MRET в данных M/A-расширений (по 1 на ELF) — инфраструктура прокси-ядра Spike, не часть расширения.

## Запуск тестов

### RV64I
```bash
./run_microtesk_rv64.sh
```

### RV64M
```bash
./run_microtesk_rv64m.sh
```

### RV64A
```bash
./run_microtesk_rv64a.sh
```

### Полный прогон (все расширения)
```bash
docker exec riscv-env sh -c '
SPIKE=/opt/riscv-isa-sim/build/spike
BUILD_DIR=/opt/riscv-isa-sim/build
ISA=rv64imafdc_zicntr_zihpm
find "$BUILD_DIR" -name "*.gcda" -delete
ELFS=$(find /opt/microtesk-riscv/output -name "*.elf")
for elf in $ELFS; do timeout 15 "$SPIKE" --isa="$ISA" "$elf" > /dev/null 2>&1; done
lcov --capture --directory "$BUILD_DIR" --output-file /tmp/microtesk_all.info --ignore-errors gcov,source
lcov --summary /tmp/microtesk_all.info
'
```

## Замечание о «падении» F/D тестов

F/D-тесты (fadd, fsub, fmul, fdiv, fmadd, fcmp, fcvt, fmin, fcvt_w, recoding) завершаются с exit 255 — это **не падение Spike**, а расхождение MicroTESK-генерируемых ожидаемых значений с реальным вычислением IEEE 754. Подробнее: [ANALYSIS_FP_CRASH_ROOT_CAUSE.md](ANALYSIS_FP_CRASH_ROOT_CAUSE.md).

Кратко: инструкции FPU выполняются корректно, gcov-покрытие собирается. Но сравнение результата с MicroTESK-ожиданием не проходит.
