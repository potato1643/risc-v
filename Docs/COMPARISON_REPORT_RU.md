# Сравнительный анализ тестовых наборов RISC-V

## 1. Архитектура тестового стенда

Тестовый стенд развёрнут в двух контейнерах Docker.

| Контейнер | Архитектура | Назначение |
|-----------|-------------|------------|
| `riscv-env` | ARM64 | RISCOF + MicroTESK |
| `compiler-lab` | x86-64 | Imperas + сборка Sail |

**Ключевой принцип:** все три тестовых набора запускаются на **одном и том же** симуляторе Spike, скомпилированном с флагами `--coverage` (GCC). Это позволяет проводить прямое сравнение покрытия кода симулятора.

Конвейер: `① Запуск тестов → ② gcov/lcov → ③ Сравнительный анализ`

---

## 2. Единая методика сбора покрытия кода (gcov/lcov)

1. Очистка счётчиков gcov: `find $BUILD_DIR -name "*.gcda" -delete`
2. Последовательный запуск всех ELF-файлов на инструментированном Spike
3. Сбор данных: `lcov --rc lcov_branch_coverage=1 --capture --directory $BUILD_DIR --output-file <suite>_coverage.info`
4. Генерация HTML-отчёта: `genhtml --rc genhtml_branch_coverage=1`

Флаг `--rc lcov_branch_coverage=1` включает сбор **веточного покрытия** (branch coverage) — каждая точка ветвления (`if`, `switch`, `?:`, циклы) отслеживается отдельно.

## 7. Выводы

1. **Единая методика работает.** Все три набора запущены на одном gcov-инструментированном Spike, получены сравнимые метрики. RISCOF FINAL — **наивысший Line Coverage (22.0% полный, 29.0% IMAFDC-фильтр)** и Branch Coverage (3.7% полный, 13.5% IMAFDC).

2. **RISCOF — безоговорочный лидер по всем метрикам.** После добавления RV32F (342 ELF) и RV32D (679 ELF), RISCOF превосходит MicroTESK по всем трём показателям: Line +3.1pp (29.0% vs 25.9%), Func +1.4pp (22.0% vs 20.6%), Branch +1.5pp (13.5% vs 12.0%). Суммарно 1,268 ELF с 98.1% прохождением.

3. **RV32F/D — ключевой фактор превосходства.** RV32F (+1.5pp) и RV32D (+2.5pp) IMAFDC-фильтр вместе дают +4.0pp. Критические исправления: `-DFLEN=32/64` для правильного развёртывания макросов и `model_test.h ALIGNMENT=3` для 8-байтового выравнивания в RV32D.

4. **RISCOF — наиболее полный по ISA-покрытию.** 1,268 ELF покрывают I, M, A, C, F, D в обеих архитектурах (RV64 + RV32). Рекомендуется как основной инструмент для валидации покрытия кода Spike.

5. **FP-тесты: RISCOF надёжен.** 1,020/1,021 F/D тестов проходят (один timeout в RV32D). MicroTESK, напротив, имеет массовые падения FP-тестов из-за несовпадения ожидаемых значений с IEEE 754.

6. **Imperas полезен для кросс-валидации и 32-битных путей.** RV32I-тесты покрывают 32-битные пути Spike. Выявлено одно расхождение с riscvOVPsim (допустимое спецификацией).

7. **Дальнейшее повышение покрытия:** основные резервы — системный режим (привилегированные инструкции, виртуальная память), Imperas расширение до других ISA, и объединение MicroTESK + RISCOF для максимального покрытия.
