# Сравнительный анализ тестовых наборов RISC-V

## Архитектура тестового стенда

Тестовый стенд развёрнут в двух контейнерах Docker с предустановленными инструментами.
Изоляция контейнеров позволяет воспроизводить результаты независимо от окружения.

### Контейнеры

| Контейнер | Архитектура | ОС | Назначение |
|-----------|-------------|-----|------------|
| `riscv-env` | ARM64 (aarch64) | Ubuntu 22.04 | RISCOF + MicroTESK |
| `compiler-lab` | x86-64 | Ubuntu 22.04 | Imperas + сборка Sail |

### Инструменты в контейнерах

**riscv-env:**
- Spike (RISC-V ISA Simulator) v1.1.0, собран с флагами `--coverage`
- Sail C simulator (эталонная модель RISC-V, сборка от 11.01.2026)
- RISCOF v1.25.3 с плагинами Spike и Sail
- riscv-isac v0.18.0 (анализатор покрытия ISA)
- RISC-V GCC toolchain (`riscv64-unknown-elf-gcc`)
- LCOV/gcov для анализа строчного покрытия

**compiler-lab:**
- Sail C simulator v0.20.1 (сборка от 01.06.2026, расширение V отключено)
- Imperas Test Suite (открытая версия)
- RISC-V GCC (`riscv64-unknown-elf-gcc` v10.2.0)
- Z3 SMT solver

## Сравнение RISCOF и Imperas (RV32I)

Оба набора тестов нацелены на RV32I — базовый набор целочисленных инструкций RISC-V
(архитектура RV32, привилегированный режим Machine). Это позволяет провести
прямое сравнение.

### 2.1 Методология тестирования

**RISCOF** (RISC-V Compliance Framework):
- Тесты компилируются из ассемблерных файлов (`.S`) с помощью RISCV GCC
- Каждый тест запускается на эталонной модели (Sail C simulator) и на тестируемом
  устройстве (Spike DUT)
- Инструмент `riscv-isac` анализирует трассу выполнения и сопоставляет фактические
  выполненные инструкции с точками покрытия (coverpoints) из CGF-файлов
- Покрытие измеряется в абсолютном количестве coverpoints

**Imperas:**
- Тесты компилируются из ассемблерных файлов (`.S`) с помощью RISCV GCC
- Каждый тест запускается, и выводится сигнатура (контрольная сумма результатов)
- Полученная сигнатура сравнивается с эталонной сигнатурой, сгенерированной
  коммерческим симулятором `riscvOVPsim`
- Результат бинарный: PASS (сигнатуры совпадают) или FAIL (различаются)

### 2.2 Масштаб тестирования

| Характеристика | RISCOF RV32I | Imperas RV32I |
|---------------|-------------|---------------|
| Количество исходных файлов | 38 `.S` | 48 `.S` |
| Измеряемый показатель | 11 305 точек покрытия | 48 сигнатур |
| Детализация | ~300 точек покрытия на инструкцию | 1 сигнатура на инструкцию/группу |
| Эталонная модель | Sail C simulator (открытая) | riscvOVPsim (коммерческая) |
| Тестируемое устройство | Spike | Sail C simulator |

### 2.4 Анализ расхождения

**I-MISALIGN_LDST-01** — единственный тест Imperas, не прошедший проверку.

Причина: поведение модели Sail при невыровненных обращениях к памяти (misaligned
load/store) отличается от поведения эталонной модели `riscvOVPsim`. Сигнатуры
(контрольные суммы результатов) не совпадают, начиная со второй строки.

Сравнение сигнатур:
```
Sail:            riscvOVPsim:
0291a1b1         91a1b1c1
f20291a1         91a1b1c1
e2f20291         91a1b1c1
ffffffff         00000001
ffffffff         00000004
...
```

Sail обрабатывает невыровненные доступы атомарно (возвращая полное слово данных),
тогда как riscvOVPsim разбивает доступ на отдельные байтовые операции. Оба
поведения допустимы спецификацией RISC-V — спецификация определяет misaligned
доступы как опциональные и допускает различные реализации.

Для устранения расхождения можно настроить Sail через параметры конфигурации
`memory.misaligned.exceptions` и `memory.misaligned.byte_by_byte`.

## Обзор всех трёх тестовых наборов

### 3.1 Сводная таблица

| Характеристика | MicroTESK | RISCOF | Imperas |
|---------------|-----------|--------|---------|
| **Архитектура** | RV64I | RV64I+M, RV32I | RV32I |
| **Количество тестов** | 30 ELF | 63 `.S` / 32 357 cp | 48 `.S` |
| **Процент прохождения** | 96.67% (29/30) | 100% | 97.92% (47/48) |
| **Строчное покрытие (gcov)** | **16.2%** | Не измеряется | Не измеряется |
| **Функциональное покрытие** | 12.2% | Не измеряется | Не измеряется |
| **Покрытие ISA** | Не применимо | CGF coverpoints | Сравнение сигнатур |
| **Эталонная модель** | Отсутствует | Sail C (открытая) | riscvOVPsim (коммерческая) |
| **Открытость** | Полностью | Полностью | Фреймворк — открыт, сигнатуры — коммерческие |
| **Сложность запуска** | Высокая (MicroTESK toolchain) | Средняя (`pip install riscof`) | Низкая (make + GCC) |

### 3.2 Сравнительный анализ по критериям

#### Критерий 1: Поддержка расширений ISA

| Расширение | MicroTESK | RISCOF | Imperas |
|-----------|-----------|--------|---------|
| RV32I | Нет | **Да** (11 305 cp) | **Да** (48 тестов) |
| RV64I | **Да** (алгоритмы) | **Да** (13 624 cp) | Нет (только док.) |
| M (умножение/деление) | Нет | **Да** (7 428 cp) | Нет |
| C (сжатые инструкции) | Нет | Да | Нет |
| F/D (плавающая точка) | Нет | Да | Нет |
| Прочие (28+ расширений) | Нет | Да | Нет |

**Вывод:** RISCOF обеспечивает на порядок более широкую поддержку расширений ISA.

#### Критерий 2: Покрытие спецификации

- **RISCOF** — единственный набор, предоставляющий количественную оценку покрытия
  спецификации через систему CGF coverpoints. 11 305–21 052 точек покрытия в
  зависимости от архитектуры, 100% покрытие.
- **Imperas** — качественная оценка через сравнение сигнатур с коммерческим
  эталоном. 48 сигнатур для RV32I.
- **MicroTESK** — не предназначен для оценки покрытия спецификации.

#### Критерий 3: Строчное покрытие кода (Code Coverage)

- **MicroTESK** — единственный набор, предоставляющий строчное покрытие через
  gcov/LCOV: 16.2% строк, 12.2% функций кода Spike. Это объясняется тем, что
  MicroTESK использует алгоритмические тесты (сортировка, деление), которые не
  покрывают все инструкции.
- **RISCOF и Imperas** — не предоставляют строчного покрытия, так как нацелены
  на покрытие ISA, а не кода симулятора.

#### Критерий 4: Прочие характеристики

| Характеристика | MicroTESK | RISCOF | Imperas |
|---------------|-----------|--------|---------|
| Тип тестов | Алгоритмические | Покадровый перебор ISA | Поинструкционные сигнатуры |
| Время выполнения | ~2 мин | ~10 мин | ~3 мин |
| Воспроизводимость | Да (Docker) | Да (Docker) | Да (Docker) |
| CI/CD интеграция | Сложно | Легко (Python CLI) | Средне (Makefile) |
| Документация | Слабая | Хорошая | Средняя |
| Анализ проваленных тестов | Лог Spike | CGF coverage report | diff сигнатур |

## Выводы

1. **RISCOF — наиболее полный набор** для верификации соответствия спецификации
   RISC-V. 100% покрытие 11 305 точек RV32I и 21 052 точек RV64I+M.
   Полностью открытый. Рекомендуется как основной инструмент.

2. **Imperas — полезен для кросс-валидации** с коммерческим эталоном riscvOVPsim.
   Ограничен RV32I в открытой версии. Обнаруженное расхождение (I-MISALIGN_LDST-01)
   не является ошибкой — оба поведения допустимы спецификацией.

3. **MicroTESK — дополнительный инструмент** для сбора строчного покрытия кода.
   Алгоритмические тесты (сортировка, деление, работа с регистрами) не
   перекрываются с поинструкционными тестами RISCOF/Imperas и могут
   использоваться для увеличения строчного покрытия симулятора.

4. **Рекомендуемая стратегия:** использовать RISCOF как базовый набор,
   дополнить Imperas для кросс-валидации и MicroTESK для сбора метрик
   строчного покрытия.
