# Сравнительный анализ тестовых наборов RISC-V

## 1. Архитектура тестового стенда

Тестовый стенд развёрнут в двух контейнерах Docker.

| Контейнер | Архитектура | Назначение |
|-----------|-------------|------------|
| `riscv-env` | ARM64 | RISCOF + MicroTESK |
| `compiler-lab` | x86-64 | Imperas + сборка Sail |

**Ключевой принцип:** все три тестовых набора запускаются на **одном и том же** симуляторе Spike, скомпилированном с флагами `--coverage` (GCC). Это позволяет проводить прямое сравнение покрытия кода симулятора.

Конвейер: `① Запуск тестов → ② gcov/lcov → ③ Сравнительный анализ`

---

## 2. Единая методика сбора покрытия кода (gcov/lcov)

1. Очистка счётчиков gcov: `find $BUILD_DIR -name "*.gcda" -delete`
2. Последовательный запуск всех ELF-файлов на инструментированном Spike
3. Сбор данных: `lcov --rc lcov_branch_coverage=1 --capture --directory $BUILD_DIR --output-file <suite>_coverage.info`
4. Генерация HTML-отчёта: `genhtml --rc genhtml_branch_coverage=1`

Флаг `--rc lcov_branch_coverage=1` включает сбор **веточного покрытия** (branch coverage) — каждая точка ветвления (`if`, `switch`, `?:`, циклы) отслеживается отдельно.

> **Ключевой вывод:** при общем знаменателе 53,227 строк, MicroTESK покрывает 98.5% всех строк, доступных трём наборам вместе. RISCOF и Imperas добавляют лишь 161 уникальную строку к MicroTESK.

## 7. Выводы

1. **Единая методика работает.** Все три набора запущены на одном gcov-инструментированном Spike, получены сравнимые метрики. MicroTESK — наивысший Line Coverage (19.6% полный, 25.9% IMAFDC-фильтр) и Branch Coverage (3.2% полный, 12.0% IMAFDC).

2. **MicroTESK — единственный набор с охватом M, A, C, F, D.** 214 ELF против 50 (RISCOF) и 48 (Imperas). Инструкциональный охват 79.7% (157/197). Дополнительные расширения дали +2.9 п.п. к Line Coverage (с 16.7% до 19.6%).

3. **Объединённое покрытие = 19.9%.** Все три набора вместе покрывают 19.9% строк (10601/53227). MicroTESK отдельно даёт 19.6% — RISCOF и Imperas добавляют лишь 161 строку (0.3 п.п.). Основной прирост — за счёт RV32I-тестов Imperas, не полностью перекрытых MicroTESK.

4. **RISCOF — наиболее полный по ISA-покрытию (CGF).** 100% 21 052 точек RV64I+M. Рекомендуется как основной инструмент для валидации соответствия ISA.

5. **Imperas полезен для кросс-валидации и 32-битных путей.** RV32I-тесты покрывают 32-битные пути Spike (+242 уникальные строки). Выявлено одно расхождение с riscvOVPsim (допустимое спецификацией).

6. **F/D тесты: покрытие собирается, но верификация не проходит.** MicroTESK генерирует ожидаемые значения, не совпадающие с IEEE 754 (Spike). Не влияет на gcov-покрытие, но ограничивает проверку правильности FPU-вычислений. Детали: [ANALYSIS_FP_CRASH_ROOT_CAUSE.md](../Тестовые%20наборы/MicroTESK/ANALYSIS_FP_CRASH_ROOT_CAUSE.md).

7. **Дальнейшее повышение покрытия:** основные резервы — системный режим (привилегированные инструкции, виртуальная память) и непокрытые инструкции RV64I (сдвиги, беззнаковые, W-операции, Zicsr).
