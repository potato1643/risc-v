# Сравнительный анализ тестовых наборов RISC-V

## 1. Архитектура тестового стенда

Тестовый стенд развёрнут в двух контейнерах Docker.

| Контейнер | Архитектура | Назначение |
|-----------|-------------|------------|
| `riscv-env` | ARM64 | RISCOF + MicroTESK |
| `compiler-lab` | x86-64 | Imperas + сборка Sail |

**Ключевой принцип:** все три тестовых набора запускаются на **одном и том же** симуляторе Spike, скомпилированном с флагами `--coverage` (GCC). Это позволяет проводить прямое сравнение покрытия кода симулятора.

Конвейер: `① Запуск тестов → ② gcov/lcov → ③ Сравнительный анализ`

---

## 2. Единая методика сбора покрытия кода (gcov/lcov)

1. Очистка счётчиков gcov: `find $BUILD_DIR -name "*.gcda" -delete`
2. Последовательный запуск всех ELF-файлов на инструментированном Spike
3. Сбор данных: `lcov --rc lcov_branch_coverage=1 --capture --directory $BUILD_DIR --output-file <suite>_coverage.info`
4. Генерация HTML-отчёта: `genhtml --rc genhtml_branch_coverage=1`

Флаг `--rc lcov_branch_coverage=1` включает сбор **веточного покрытия** (branch coverage) — каждая точка ветвления (`if`, `switch`, `?:`, циклы) отслеживается отдельно.

## 8. Выводы

1. **Единая методика работает.** Все три набора запущены на двух gcov-инструментированных симуляторах (Spike и Sail C), получены сравнимые метрики. RISCOF FINAL — **наивысший Line Coverage на Spike (22.0% полный, 29.0% IMAFDC-фильтр)**.

2. **RISCOF — безоговорочный лидер на Spike.** После добавления RV32F/D, RISCOF превосходит MicroTESK по всем трём показателям: Line +3.1pp, Func +1.4pp, Branch +1.5pp. Суммарно 1,268 ELF с 98.1% прохождением.

3. **MicroTESK лидирует на Sail по эффективности.** На Sail C-симуляторе MicroTESK (214 ELF) показывает **28.9% C-уровень** и **7.3% спецификацию Sail** — опережая RISCOF RV32 (1,103 ELF: 28.3% и 7.1%). Шаблонные тесты MT эффективнее затрагивают монолитный код Sail.

4. 🆕 **RV64+RV32 — максимум RISCOF на Sail: 29.2% C, 7.8% спецификация.** RV64 IMAFDC (159 ELF, 27.6%) вносит +222 уник. точки спецификации к RV32 (2,761 → 2,983). RV64 лидирует в I/M/A/C, но RV32 доминирует в F/D благодаря 1,021 FP тестам.

5. 🆕 **Покрытие спецификации Sail — новый рубеж.** Впервые собран spec-level coverage (5.0-7.8%) наряду с C-код покрытием (25-29%). Покрытие спецификации стабильно составляет ~20-25% от C-покрытия: формальная спецификация гораздо полнее сгенерированного C-кода.

6. 🔄 **IMAFDC + Privileged = 9.7% спецификации Sail.** Privileged (204 ELF) добавляет +754 уник. точки (+1.9pp) — наибольший вклад отдельного компонента. PMP (65 ELF) — самый эффективный: 2,590 точек (6.7% в одиночку). Впервые превышен 9% рубеж.

7. 🆕 **>90% спецификации Sail не покрыто.** Даже RISCOF IMAFDC+Privileged (9.7%) оставляет непокрытыми >90% формальной спецификации. Это фундаментальный разрыв между конформным тестированием и верификацией спецификации.

8. **FP-тесты: RISCOF надёжен.** 1,020/1,021 F/D тестов проходят. MicroTESK имеет падения FP-тестов из-за несовпадения ожидаемых значений с IEEE 754.

9. **Imperas полезен для кросс-валидации.** RV32I-тесты покрывают базовые пути. На Sail: 5.0% спецификации при всего 48 ELF.

8. **Дальнейшие шаги:** перезапуск RV64 на build_sailcov для консистентности, итоговый отчёт, фильтр IMAFDC для Sail спецификации.
